# 生产级交付编排器

## 强制加载完整交付协议

在制定计划、修改文件或执行任务专属操作之前，必须完整读取 `references/system-prompt.md`。

用户明确选择保留完整的长规则。不得用摘要替代它，不得只读取其中几个章节。

将该文件视为当前任务的稳定交付契约。它始终低于平台和系统指令、用户当前请求以及适用的项目指令文件。

当最终结果、用户旅程、验收门槛或授权的交付层级存在实质性歧义时，读取 `references/outcome-contract.md`。

## 执行老板式结果门

在实现前提取：

- 用户真正想解决的问题；
- 用户、运营人员或调用方最终可以观察到的状态；
- 从入口、操作到结果的完整用户旅程；
- 必须满足的 MUST 验收标准；
- 允许范围以及必须保留的旧行为；
- 用于证明交付完成的真实验证证据；
- 外部写入、破坏性操作、付费调用、生产操作和远程操作边界。

如果这些信息已经清晰，直接继续，不要让用户重复回答。

如果缺失的选择会实质改变产品行为、交付范围、架构、数据、成本、风险或可逆性，执行最小必要的一轮提问：

- 每轮只询问一到三个简短问题；
- 每个问题提供两到三个互斥的结果选项；
- 将推荐选项放在第一个，并明确标注“推荐”；
- 说明用户最终会获得或看到什么，不要只列出技术术语；
- 环境提供结构化询问工具时优先使用，否则用普通文本询问；
- 如果客户端会自动提供“其他”选项，不要重复添加。

除非技术选择会带来真实的业务、产品、成本、风险或兼容性取舍，否则由智能体根据仓库事实决定技术实现。让用户像产品负责人一样选择结果，不要迫使用户决定库、文件名和内部分层。

对不阻塞的歧义，采用最安全、可逆的推荐假设，简短说明后继续。当选择涉及不可逆操作、生产环境、外部写入、真实费用、敏感数据或实质性扩大范围时，必须等待明确授权。

结果对齐后，将其转换为简洁的内部任务契约：

```text
目标：
用户旅程：
MUST 验收标准：
允许范围：
必须保留：
明确不做：
验证证据：
审批边界：
停止条件：
```

如果可以从上下文可靠推导，不要强迫用户填写这份模板。

## 执行生产级交付循环

1. 识别用户只授权分析，还是同时授权修改和实现。
2. 使用完整交付协议将任务分类为 Quick、Standard 或 Deep。
3. 检查适用的项目指令、当前工作区状态、架构、现有模式、测试以及相关工具或技能。
4. 根据风险和依赖关系决定计划深度，不对简单任务制造流程文档。
5. 完成当前范围内的真实行为，包括受影响的上下游接缝、失败状态和边界情况。
6. 执行最相关的真实验证。不得把未执行的检查声称为通过。
7. 根据任务风险完成审查、盲点扫描、文档同步和最终验收。
8. 只要仍有未满足的 MUST 标准或未解决的 P0/P1 问题，并且下一轮安全、在范围内且能产生实质进展，就继续处理。
9. 当交付契约已满足，或出现必须由用户授权、补充信息或改变外部状态才能解决的阻塞时停止。

最终回答先给出交付结论，再保留已执行的验证证据、剩余边界和需要批准的下一步。删除重复背景和无价值的工具调用旁白。

## 跨智能体安装

本目录遵循通用 `SKILL.md` Agent Skills 结构。当用户要求安装或生成项目规则桥接时，可使用 `scripts/install_skill.py`：

```bash
python scripts/install_skill.py --scope user --targets all
python scripts/install_skill.py --scope project --project-dir /path/to/repo --targets all --bridges all
python scripts/install_skill.py --scope project --project-dir /path/to/repo --custom-bridge .tool/rules.md
```

原生安装目标包括 Codex、Claude Code 和通用 Agent Skills 目录。项目桥接覆盖 `AGENTS.md`、`CLAUDE.md`、`GEMINI.md`、GitHub Copilot 指令、Cursor 规则、Windsurf 规则和 Cline 规则。其他支持项目指令文件的 CLI 或 IDE 可使用 `--custom-bridge`。

团队共享仓库优先使用项目级安装；个人所有项目默认启用时使用用户级安装。除非用户明确授权 `--force`，否则不要覆盖现有安装。
